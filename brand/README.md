# AIRED brand kit

The **AIRED.** lockup, reproduced exactly from the site's landing/onboarding
mark (`src/components/onboarding/onboarding.tsx` — `ScreenOne`): the wordmark
**AIRED** in a clean geometric sans with wide letter-spacing, a **red period**
after the D, and a short **glowing red underline**, all flat on matte black.
No metallic, no bevel, no chrome — restraint is the point.

Everything here is generated by `generate.mjs` — re-run it any time:

```bash
node brand/generate.mjs
```

Zero dependencies (Node built-ins only), no network, no fonts.

## The lockup

- **Wordmark** — `AIRED`, off-white, wide tracking (~0.2em), medium geometric
  weight. Drawn as **hand-built geometric vector paths** (thickened skeletons of
  segments, elliptical arc-strokes and disks) — no font is downloaded or
  embedded, so the build is fully self-contained.
- **Red period** — a filled red dot immediately after the D. The signature; it
  is always present (never a lone letter).
- **Red underline** — a short, thin, centred bar (~40% of the lockup width) with
  a soft red glow, echoing the platform's Red Line.

## Palette (pixel-verified)

| Token | Hex | Use |
|-------|-----|-----|
| Matte black | `#0a0a0a` | background field |
| Off-white | `#ededed` | wordmark |
| Signal red | `#e10600` | period, underline, glow |

> **Note on the red.** The kit uses **`#e10600`** (the spec's "signal red"). The
> live site currently sets `--cert-red: #ff2d2d` (a brighter glow-red). To match
> the site exactly instead, change `RED` at the top of `generate.mjs` to
> `#ff2d2d` and re-run — it's a single constant.

## How it's built

`generate.mjs` defines the mark as geometric **contours** (closed polygons in
device pixels) and renders them two ways that agree pixel-for-pixel:

- **SVG** — each contour becomes a `<path>` (plus a `<circle>` period, a rounded
  `<rect>` underline, and a `feGaussianBlur` glow). Pure vector, editable.
- **PNG** — a small nonzero-winding **scanline rasterizer** (`lib/render.mjs`)
  fills the same contours with anti-aliasing, adds the glow via a separable box
  blur, and encodes the result with Node's built-in `zlib`.

`lib/glyphs.mjs` holds the hand-built letterforms and the text layout used for
the wordmark and the tagline.

On every run the generator asserts each output's exact dimensions and samples
the background, wordmark, period and underline pixels against the palette,
printing a pass/fail report and exiting non-zero if anything drifts.
